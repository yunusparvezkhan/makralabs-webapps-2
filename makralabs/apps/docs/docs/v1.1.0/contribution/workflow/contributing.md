# Contributing

Makra documentation and SDK contributions should be scoped, tested, and written with the user task in mind.

## Before opening a pull request

- Confirm the problem statement.
- Add or update tests where behavior changed.
- Document new configuration, endpoints, or workflow changes.

## Pull request checklist

1. Explain the user-facing problem.
2. Describe the implementation at a high level.
3. Call out risks or migrations.
4. Include screenshots or examples when behavior changed.

## Review philosophy

Smaller pull requests move faster and are easier to validate than broad refactors.
