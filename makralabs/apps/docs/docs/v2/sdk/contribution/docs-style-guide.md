# Docs Style Guide

Documentation should help readers complete work quickly, not make them reverse-engineer intent from vague prose.

## Writing principles

- Lead with the task the page helps complete.
- Put prerequisites before commands.
- Prefer short paragraphs and explicit examples.
- Separate reference facts from tutorials.

## Code sample guidance

- Use realistic variable names.
- Show the smallest useful example first.
- Include expected output when it helps validate success.

## Information architecture

Place setup guides early, keep reference pages stable, and move exploratory material into examples or release notes until it settles.
