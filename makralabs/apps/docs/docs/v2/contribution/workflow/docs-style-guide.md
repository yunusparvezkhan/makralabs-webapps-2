# Docs Style Guide

Good product documentation helps readers complete work quickly without reverse-engineering intent.

## Writing principles

- Lead with the task the page supports.
- Put prerequisites before commands.
- Prefer short paragraphs and concrete examples.
- Separate tutorials from stable reference material.

## Code sample rules

- Use realistic variable names.
- Show the smallest useful example first.
- Include expected output when it clarifies success.

## Structure guidance

Keep top-level navigation stable and move evolving material into examples until it is ready for reference documentation.
