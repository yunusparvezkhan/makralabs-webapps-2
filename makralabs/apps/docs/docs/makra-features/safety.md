# Safety & governance

Makra is designed for production AI systems where data boundaries matter.

## Redaction

Apply redaction before data is stored:

- API keys
- Access tokens
- Personal identifiers (as required by your policy)

## Allowlists

If you let users connect external sources (docs, web pages, drives):

- Maintain an allowlist of domains and sources
- Store provenance for every snippet
- Provide citations in user-facing answers

## Data lifecycle

Define retention rules:

- Session expiration
- Document deletion
- “Right to be forgotten” workflows
